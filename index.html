Einführung
Was ist Echtzeit-Objekterkennung?

Echtzeit-Objekterkennung ist die automatische Identifikation und Lokalisierung von Objekten in Bildern oder Videoströmen mit minimaler Verzögerung. Im Gegensatz zu klassischen Bildanalyseverfahren ermöglicht sie die gleichzeitige Erkennung mehrerer Objekte und deren Klassifizierung in wenigen Millisekunden.

Die Technologie nutzt tiefe neuronale Netze (Deep Learning), um visuell Muster zu erkennen und präzise Bounding Boxes um Objekte zu zeichnen – Grundlage für autonomes Fahren, Videoüberwachung, medizinische Bilddiagnose und industrielle Qualitätsprüfung.

Kurz erklärt
Was ist Echtzeit-Objekterkennung?

Objekterkennung löst gleichzeitig zwei Aufgaben: Lokalisierung (wo ist das Objekt? → Bounding Box) und Klassifikation (was ist es? → Klasse + Konfidenz). Echtzeit bedeutet mindestens 25–30 Frames pro Sekunde – jeder Frame muss in unter 33 ms verarbeitet sein.

📦

Lokalisierung

Position und Größe jedes Objekts als Bounding Box (x, y, Breite, Höhe).

🏷️

Klassifikation

Gleichzeitige Zuweisung eines Klassen-Labels mit Konfidenzwert.

⚡

Echtzeitfähigkeit

One-Stage-Detektoren wie YOLO erreichen >30 FPS auf Consumer-Hardware.

~2000 Regionenvorschläge → CNN → SVM. Durchbruch für CNN-Detektoren, aber noch zu langsam für Echtzeit (47 s/Bild).

Meilenstein

2023
Aktuell

Ankerfreie Architektur, 37,3 mAP auf COCO, einheitliche API für Detection/Segmentation. Nano-Variante: 3,2 ms/Bild auf A100.

Begriffserklärungen
Begriff 01
Bounding Box

Achsenparalleles Rechteck um ein erkanntes Objekt – definiert durch Mittelpunkt (cx, cy) sowie Breite und Höhe.

Begriff 02
Confidence Score

Zahl [0–1]: Wie sicher ist das Modell, dass die Box ein Objekt enthält und die Klasse korrekt ist?

Begriff 03
IoU – Intersection over Union

Schnittfläche / Vereinigungsfläche zweier Boxen. 0 = keine Überlappung, 1 = identisch. COCO-Schwellwert: ≥ 0,5.

Begriff 04
NMS – Non-Maximum Suppression

Bereinigt mehrfach erkannte Objekte: Beste Box behalten, alle Nachbarboxen mit IoU > 0,45 unterdrücken.

Begriff 05
Anchor Box

Vordefinierte Box-Formen als Vorhersage-Referenz. YOLOv8 ist ankerfrei und sagt den Mittelpunkt direkt vorher.

Begriff 06
mAP – mean Average Precision

Hauptmetrik für Objektdetektoren: Durchschnitt der Average Precision über alle Klassen bei variierenden IoU-Schwellwerten.

Kapitel 2
Motivation

Warum ist Geschwindigkeit so kritisch? Ein autonomes Fahrzeug, das einen Fußgänger 200 ms zu spät erkennt, hat bei 50 km/h bereits 2,8 m zurückgelegt – zu viel für eine sichere Reaktion. Millisekunden entscheiden hier über Leben und Tod, und das gilt nicht nur im Straßenverkehr: Industrieroboter, die Defekte am Fließband erkennen, medizinische Systeme, die während einer Operation assistieren, oder Sicherheitskameras, die Eindringlinge in Echtzeit melden – all diese Anwendungen verlangen, dass jeder einzelne Frame innerhalb von unter 33 ms vollständig verarbeitet ist.

Der Bedarf wächst rasant: Der globale Markt für Computer Vision wird bis 2030 auf über 50 Milliarden US-Dollar geschätzt. Was vor zehn Jahren noch spezialisierte Rechenzentren erforderte, läuft heute auf Smartphones, Mikrocontrollern und direkt im Browser – und macht Echtzeit-Objekterkennung damit zu einer der praktisch relevantesten Technologien unserer Zeit.

🐢 Two-Stage – langsam

Faster R-CNN erzielt hohe Genauigkeit (~42 mAP), benötigt aber 100–500 ms pro Frame – für Live-Video nicht einsetzbar.

⚡ One-Stage – schnell

YOLOv8n erreicht 37 mAP bei unter 5 ms pro Frame auf der GPU, im Browser 10–30 FPS – Echtzeit ganz ohne Server.

Rolle von OpenCV für Video-Capture und Vorverarbeitung

OpenCV übernimmt in klassischen Pipelines Video-Capture, Farbkonvertierung (BGR→RGB), Resize/Letterboxing und die Visualisierung der Bounding Boxes. In dieser Browser-Demo erfüllt JavaScript + Canvas dieselbe Aufgabe.

OpenCV + YOLO (Python) 
import cv2
from ultralytics import YOLO

model = YOLO('yolov8n.pt')
cap   = cv2.VideoCapture('video.mp4')  # oder 0 für Webcam

while cap.isOpened():
    ret, frame = cap.read()
    if not ret: break
    results = model.predict(frame, conf=0.30)
    cv2.imshow('YOLO', results[0].plot())
    if cv2.waitKey(1) == ord('q'): break

One-stage Detector · Diese Demo

You Only Look Once

YOLO teilt das Bild in ein S×S-Gitter. Jede Zelle sagt gleichzeitig mehrere Bounding-Boxes mit Konfidenz-Scores und Klassenwahrscheinlichkeiten voraus – alles in einem einzigen Vorwärtsdurchlauf durch das CNN. Das macht YOLO ideal für Echtzeitanwendungen.

Diese Demo verwendet YOLOv8n (nano), das direkt im Browser per ONNX Runtime WebAssembly läuft – kein Server erforderlich. Das Modell ist auf dem COCO-Datensatz mit 80 Klassen vortrainiert und demonstriert Transfer Learning in der Praxis.

YOLOv8n · ONNX 640×640 Input 80 COCO-Klassen WebAssembly Client-seitig
Funktionsweise
Wie YOLO grob funktioniert
  • Input Ein Bild beliebiger Größe wird auf ein festes Format normalisiert.
  • Backbone-Netzwerk Das tiefe neuronale Netz extrahiert Merkmale aus dem Bild in mehreren Schichten.
  • Gitter-Struktur Das Bild wird in Zellen aufgeteilt. Jede Zelle ist für Objekte in ihrem Bereich zuständig.
  • Vorhersage Für jede Zelle werden Bounding Boxes, Objectness Score und Klassenwahrscheinlichkeiten vorhergesagt.
  • Output Kombination aller Vorhersagen ergibt die finalen Bounding Boxes mit Klassifizierung.
  • Post-Processing NMS eliminiert doppelte Erkennungen und filtert schwache Detektionen.
🖼️
Rolle von OpenCV OpenCV (Open Source Computer Vision) übernimmt die Bildvorverarbeitung: Skalierung und Normalisierung für das Netzwerk, Farbkonvertierung (RGB ↔ BGR) sowie das Zeichnen von Bounding Boxes und Labels auf das Ausgabebild.
