Was ist Echtzeit-Objekterkennung?

Objekterkennung löst gleichzeitig zwei Aufgaben: Lokalisierung (wo ist das Objekt? → Bounding Box) und Klassifikation (was ist es? → Klasse + Konfidenz). Echtzeit bedeutet mindestens 25–30 Frames pro Sekunde – jeder Frame muss in unter 33 ms verarbeitet sein.

📦

Lokalisierung

Position und Größe jedes Objekts als Bounding Box (x, y, Breite, Höhe).

🏷️

Klassifikation

Gleichzeitige Zuweisung eines Klassen-Labels mit Konfidenzwert.

⚡

Echtzeitfähigkeit

One-Stage-Detektoren wie YOLO erreichen >30 FPS auf Consumer-Hardware.

Geschichte der Objekterkennung
2001
Klassisch

Viola-Jones – Erster Echtzeit-Detektor

Haar-Merkmale + Cascade Classifier ermöglichen erstmals Echtzeit-Gesichtserkennung auf Consumer-Hardware.

2012
Durchbruch

AlexNet – Deep Learning Revolution

Tiefes CNN gewinnt ImageNet und halbiert die Fehlerrate. GPU-Training und Deep Learning werden zum Standard.

2013
CNN

R-CNN – Regionenbasierte Erkennung

~2000 Regionenvorschläge → CNN → SVM. Durchbruch für CNN-Detektoren, aber noch zu langsam für Echtzeit (47 s/Bild).

2015
Meilenstein

YOLO v1 – You Only Look Once

Ein einziger Forward-Pass liefert alle Bounding Boxes direkt. 45 FPS auf einer GPU – Echtzeit-Objekterkennung ist Realität.

2023
Aktuell

YOLOv8 – Stand der Technik

Ankerfreie Architektur, 37,3 mAP auf COCO, einheitliche API für Detection/Segmentation. Nano-Variante: 3,2 ms/Bild auf A100.

Begriffserklärungen
Begriff 01
Bounding Box

Achsenparalleles Rechteck um ein erkanntes Objekt – definiert durch Mittelpunkt (cx, cy) sowie Breite und Höhe.

Begriff 02
Confidence Score

Zahl [0–1]: Wie sicher ist das Modell, dass die Box ein Objekt enthält und die Klasse korrekt ist?

Begriff 03
IoU – Intersection over Union

Schnittfläche / Vereinigungsfläche zweier Boxen. 0 = keine Überlappung, 1 = identisch. COCO-Schwellwert: ≥ 0,5.

Begriff 04
NMS – Non-Maximum Suppression

Bereinigt mehrfach erkannte Objekte: Beste Box behalten, alle Nachbarboxen mit IoU > 0,45 unterdrücken.

Begriff 05
Anchor Box

Vordefinierte Box-Formen als Vorhersage-Referenz. YOLOv8 ist ankerfrei und sagt den Mittelpunkt direkt vorher.

Begriff 06
mAP – mean Average Precision

Hauptmetrik für Objektdetektoren: Durchschnitt der Average Precision über alle Klassen bei variierenden IoU-Schwellwerten.

Kapitel 2
Motivation

Warum ist Geschwindigkeit so kritisch? Ein autonomes Fahrzeug, das einen Fußgänger 200 ms zu spät erkennt, hat bei 50 km/h bereits 2,8 m zurückgelegt – zu viel für eine sichere Reaktion. One-Stage-Detektoren wie YOLO lösen den Zielkonflikt zwischen Genauigkeit und Geschwindigkeit.

🐢 Two-Stage langsam

Faster R-CNN: hohe Genauigkeit (~42 mAP), aber 100–500 ms/Frame – zu langsam für Live-Video.

⚡ One-Stage schnell

YOLOv8n: 37 mAP bei <5 ms/Frame auf GPU, im Browser 10–30 FPS – Echtzeit auch ohne Server.

Rolle von OpenCV für Video-Capture und Vorverarbeitung

OpenCV übernimmt in klassischen Pipelines Video-Capture, Farbkonvertierung (BGR→RGB), Resize/Letterboxing und die Visualisierung der Bounding Boxes. In dieser Browser-Demo erfüllt JavaScript + Canvas dieselbe Aufgabe.

OpenCV + YOLO (Python) 
import cv2
from ultralytics import YOLO

model = YOLO('yolov8n.pt')
cap   = cv2.VideoCapture('video.mp4')  # oder 0 für Webcam

while cap.isOpened():
    ret, frame = cap.read()
    if not ret: break
    results = model.predict(frame, conf=0.30)
    cv2.imshow('YOLO', results[0].plot())
    if cv2.waitKey(1) == ord('q'): break

Lokale Demo mit OpenCV + YOLO

scripts/yolo_demo.py führt YOLOv8-Inferenz lokal aus und speichert ein annotiertes Ergebnisbild in runs/detect/.

scripts/yolo_demo.py 
from ultralytics import YOLO

model   = YOLO("yolov8n.pt")
results = model.predict("input.mp4", conf=0.30, save=True)
print(f"Ergebnisse gespeichert in: {results[0].save_dir}")
