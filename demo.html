Live-Demo: Video-Upload mit Erkennung

Lade eine beliebige Videodatei hoch – YOLOv8n erkennt Objekte in jedem Frame in Echtzeit. Das Modell läuft vollständig im Browser via ONNX Runtime WebAssembly. Passe den Konfidenz-Schwellwert an, um Precision und Recall zu beeinflussen.

🎬 Videodatei hier ablegen oder klicken zum Auswählen MP4 · WebM · MOV · AVI
Modell wird geladen … 0.30
💡
Tipp: Konfidenz-Schwellwert Niedrigerer Wert (z. B. 0,15) → mehr Detektionen, höheres Recall, mehr Fehlalarme. Höherer Wert (z. B. 0,70) → nur sichere Detektionen, höhere Precision, möglicherweise werden schwache Objekte übersehen. Der Standard von 0,30 ist ein guter Ausgangspunkt.

YOLO vs. TensorFlow Object Detection API

Neben YOLO ist die TensorFlow Object Detection API (TF OD API) eines der meistgenutzten Frameworks für Objekterkennung. Beide Ansätze haben unterschiedliche Stärken – die Wahl hängt von Anforderungen und Kontext ab.

Merkmal YOLOv8n (diese Demo) TF Object Detection API
Architektur One-Stage (ankerfrei) Two-Stage (Faster R-CNN) oder One-Stage (SSD, EfficientDet)
Framework PyTorch (Ultralytics) TensorFlow / Keras
Inferenz-Geschwindigkeit ~3–30 ms/Frame ~50–500 ms/Frame (Two-Stage)
mAP auf COCO 37,3 @ [0.5:0.95] (Nano) ~28–45 (je nach Modell: SSD bis Faster R-CNN)
Modellgröße ~6 MB (nano) ~25–200 MB
Browser-fähig Ja (ONNX Runtime Web) Begrenzt (TensorFlow.js)
Training/Finetuning Sehr einfach (1 Zeile) Komplex (Config-Dateien, Pipeline)
Export-Formate ONNX, TFLite, CoreML, TensorRT SavedModel, TFLite, TF.js
Community & Docs Sehr aktiv (Ultralytics) Groß (Google-backed)
Lizenz AGPL-3.0 (kommerziell: kostenpflichtig) Apache 2.0 (frei)
Beste Eignung Echtzeit, Edge, Rapid Prototyping Forschung, Produktionspipelines, TF-Ökosystem
⚖️
Wann welche Bibliothek? YOLO ist erste Wahl, wenn Geschwindigkeit und einfaches Deployment wichtig sind. Die TF OD API ist besser integriert, wenn ohnehin mit TensorFlow gearbeitet wird oder wenn komplexe Two-Stage-Modelle für maximale Genauigkeit nötig sind. Für diese Browser-Demo war YOLO mit ONNX-Export die einzig praktikable Option.
Lokal ausführen
Lokale Demo mit OpenCV + YOLO

Das Skript scripts/yolo_demo.py führt YOLOv8-Inferenz auf einem Bild oder Video lokal aus und speichert ein annotiertes Ergebnisbild. So lässt sich die Demo ohne Browser und mit voller GPU-Beschleunigung reproduzieren.

📦 Voraussetzungen

Python 3.8+ und folgende Pakete:

pip install ultralytics opencv-python

Das Modell yolov8n.pt wird beim ersten Aufruf automatisch von Ultralytics heruntergeladen (~6 MB).

▶️ Ausführung

# Einzelbild
python scripts/yolo_demo.py --input bild.jpg

# Video
python scripts/yolo_demo.py --input video.mp4

# Webcam (live)
python scripts/yolo_demo.py --input 0
scripts/yolo_demo.py 
"""
Lokale YOLO-Demo: Inferenz auf Bild, Video oder Webcam.
Ergebnisbild/-video wird in runs/detect/ gespeichert.
"""
import argparse
import cv2
from ultralytics import YOLO

def run(source: str, conf: float = 0.30, save: bool = True):
    model = YOLO("yolov8n.pt")

    if source == "0" or source.isdigit():
        # Webcam: Ergebnisse live anzeigen
        for result in model.predict(int(source), stream=True, conf=conf):
            cv2.imshow("YOLO Demo", result.plot())
            if cv2.waitKey(1) == ord("q"):
                break
        cv2.destroyAllWindows()
    else:
        # Bild oder Video: Ergebnisse speichern
        results = model.predict(source, conf=conf, save=save)
        print(f"Gespeichert in: {results[0].save_dir}")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--input", required=True)
    parser.add_argument("--conf", type=float, default=0.30)
    args = parser.parse_args()
    run(args.input, args.conf)
🖼️
Ergebnisbild Nach Ausführung speichert Ultralytics das annotierte Bild automatisch in runs/detect/predict/. Für Videos wird eine annotierte .mp4-Datei erzeugt – Bounding Boxes, Klassenlabels und Konfidenzwerte werden direkt eingebettet.
