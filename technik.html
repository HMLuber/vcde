Schritt 1
Eingabeverarbeitung: Letterboxing

YOLO erwartet quadratische Eingaben (640×640 Pixel). Um Verzerrungen beim Skalieren beliebiger Videoformate zu vermeiden, wird Letterboxing eingesetzt: Das Bild wird proportional skaliert und mit grauen Balken (Farbwert 114) aufgefüllt. Anschließend werden die Pixelwerte von [0, 255] auf [0, 1] normiert und die Kanalreihenfolge von BGR (OpenCV-Standard) auf RGB umgestellt.

📐 Skalierung

Skalierungsfaktor s = min(640/W, 640/H). Das Bild wird auf nw × nh skaliert und mittig im 640×640-Canvas platziert. Die Randpixelwerte werden gespeichert, um Bounding-Box-Koordinaten später rückzurechnen.

🎨 Normalisierung

Pixelwerte werden durch 255 dividiert (Float32). Die drei Kanäle werden in separate Ebenen separiert: erst alle Rot-, dann Grün-, dann Blauwerte (CHW-Format statt HWC), da PyTorch/ONNX dieses Layout erwartet.

Schritt 2 + 3
Backbone & Neck: Merkmalspyramide

Der Backbone (CSPDarknet) ist ein tiefes CNN, das hierarchische Merkmale extrahiert: einfache Kanten in frühen Schichten, komplexe Strukturen in späteren. Der Neck kombiniert Features verschiedener Tiefe mittels Feature Pyramid Network (FPN) und Path Aggregation Network (PAN), um sowohl kleine als auch große Objekte gut zu erfassen.

🧠
Multi-Scale-Erkennung YOLOv8 gibt Vorhersagen auf drei verschiedenen Auflösungsstufen aus: 80×80 (kleine Objekte), 40×40 (mittelgroße) und 20×20 (große Objekte). Jede Zelle in diesen Gittern sagt Bounding Boxes für Objekte in ihrem Bildausschnitt vorher.
Schritt 4
Detection Head: Vorhersage

Der Detection Head erzeugt für jede Gitterzelle auf jeder Skala eine Vorhersage: vier Koordinaten für die Bounding Box (cx, cy, w, h) und einen Klassifikationsvektor mit 80 Klassen-Scores (COCO-Datensatz). YOLOv8 ist ankerfrei – der Mittelpunkt der Box wird direkt relativ zur Gitterzelle vorhergesagt, was das Training vereinfacht.

Schritt 5
Non-Maximum Suppression

Das rohe Ausgabetensor enthält Tausende von Kandidat-Boxen, von denen die meisten dasselbe Objekt beschreiben. NMS bereinigt diese: Sortiere alle Boxen nach Score; behalte die beste; unterdrücke alle, die einen IoU-Overlap von >0,45 mit der besten Box haben. Wiederhole mit der nächstbesten unmarkierten Box.

⚙️
Parametereinfluss in dieser Demo Der Konfidenz-Slider steuert, welche Kandidaten überhaupt bis zur NMS-Stufe gelangen. Niedrigerer Schwellwert → mehr Kandidaten → mehr erkannte Objekte, aber auch mehr Fehldetektionen. Der IoU-Schwellwert für NMS ist in dieser Demo auf 0,45 fixiert.
Vergleich
Two-Stage vs. One-Stage Detektoren

Die Architekturwahl zwischen Two-Stage und One-Stage ist der wichtigste Designentscheid bei Objektdetektoren – sie bestimmt den Kompromiss zwischen Genauigkeit und Geschwindigkeit.

Eigenschaft Two-Stage (R-CNN-Familie) One-Stage (YOLO / SSD)
Prinzip 1. Regionenvorschläge → 2. Klassifikation Direkte Vorhersage in einem Pass
Genauigkeit Hoch Gut
Geschwindigkeit Langsam (100–500 ms) Sehr schnell (<10 ms)

Beispiele R-CNN, Fast R-CNN, Faster R-CNN, Mask R-CNN YOLO v1–v8, SSD, RetinaNet
Bibliothek
Rolle von OpenCV für Video-Capture und Vorverarbeitung

OpenCV (Open Source Computer Vision Library) ist die Standardbibliothek für Video- und Bildverarbeitung in Python und C++. In klassischen YOLO-Pipelines übernimmt OpenCV den gesamten Teil vor und nach der eigentlichen Inferenz.

📽️

VideoCapture

cv2.VideoCapture(0) öffnet eine Webcam, VideoCapture('video.mp4') eine Datei. cap.read() liefert Frame für Frame – OpenCV übernimmt Codec-Dekodierung, Pufferung und Timing automatisch.

🔧

Vorverarbeitung

OpenCV liest Bilder standardmäßig als BGR. Für YOLO-Inferenz nötig: cv2.cvtColor(frame, cv2.COLOR_BGR2RGB), anschließend Resize auf 640×640 und Normalisierung. Ultralytics' YOLO macht das intern automatisch.

🖊️

Visualisierung

Bounding Boxes werden mit cv2.rectangle() und cv2.putText() direkt auf den Frame gezeichnet. cv2.imshow() zeigt das Ergebnis live. VideoWriter speichert das annotierte Video.

🌐

In dieser Demo

Im Browser ersetzt ein HTML5-Canvas die OpenCV-Visualisierung. Video-Capture läuft über die File-API des Browsers. Vorverarbeitung und Letterboxing sind in JavaScript reimplementiert – das Prinzip ist identisch.

OpenCV + YOLO Pipeline (Python) 
import cv2
from ultralytics import YOLO

model = YOLO('yolov8n.pt')
cap   = cv2.VideoCapture('video.mp4')  # oder 0 für Webcam

while cap.isOpened():
    ret, frame = cap.read()
    if not ret: break

    # Inferenz – OpenCV BGR wird intern zu RGB konvertiert
    results = model.predict(frame, conf=0.30, iou=0.45)

    # Annotiertes Bild zurückbekommen
    annotated = results[0].plot()

    cv2.imshow('YOLO Live', annotated)
    if cv2.waitKey(1) == ord('q'): break

cap.release()
cv2.destroyAllWindows()
